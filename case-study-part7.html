Case Study Part 7

This section presents Case Study Part 7: Web Server.

Requirements

  1. Create a simple university website hosted on IIS 7.
  2. Create a simple website for each department (subdirectories in the wwwroot folder).

Steps

  1. Install Server Roles
    • Add the Web Server Role.
  2. Add Web Server Features
    • Application Development
    • IIS 6 Management Compatibility
    • FTP Server
  3. Create and Test Web Application
    • Download and install: AnyToISO Converter.
    • Start → Administrative Tools → IIS Manager.
    • Expand domain name → Right click Sites → Add Website.
    • Site Name: tupt
    • Physical Path: C:\inetpub\wwwroot\tupt
    • Assigned Address: 172.31.0.31
    • Host Name: tupt.edu.ph
  4. Configure Output Caching
    • Select site “tupt” → Output Caching.
    • Add file extension: .aspx.
    • Enable User Mode Caching.
  5. Add Content to Site
    • Go to C:\inetpub\wwwroot.
    • Move iisstart.html into the tupt folder.
    • Create subfolders for Academics, Administrative, and Department content.
    • iisstart.html serves as the index file.
  6. Access Website
    • In IIS Manager → Sites → Browse “tupt”.
    • Website should open in Internet Explorer.
    • Test by typing http://tupt.edu.ph/ in Internet Explorer on Windows XP and Windows 7 clients.
