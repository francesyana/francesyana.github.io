File Server Configuration

This section presents Installing and Configuring File Services in Windows Server 2008 to establish a centralized storage system for domain users. A file server allows multiple users to store, access, and manage files securely, while administrators control permissions and enforce file restrictions. File servers are widely used in schools, companies, and organizations to manage shared resources efficiently.

Tasks Performed

  • Install the File Services Role.
  • Create shared folders with specific permissions and file restrictions.
  • Configure File Services to manage shared folders and permissions.

Created Shared Folders

  • Public Folder
    • Permissions: Read / Write
    • Users: All
    • Allowed Files: All except *.exe, *.com, *.vb, *.vbs
  • VB.Net Folder
    • Permissions: Read / Write
    • Allowed Files: *.zip, *.rar
  • C_Programs Folder
    • Permissions: Read / Write
    • Allowed Files: *.c, *.cpp
  • Surname_Section Folder
    • Permissions: Full Control
    • Users: Domain Users
    • Allowed Files: *.c, *.cpp, *.pdf, *.rar, *.docx, *.ppt, *.dwg
  • PHOTO_Male & PHOTO_Female
    • Permissions: Read / Write
    • Access based on user gender
    • Allowed Files: *.jpg, *.gif, *.png
