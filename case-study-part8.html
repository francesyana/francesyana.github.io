Case Study Part 8

This section presents Case Study Part 8: Group Policy Management.

Requirements

  1. Each department should have its own desktop wallpaper.
  2. Restrict/disable specific features and enforce security policies using Group Policy Objects (GPOs).

Steps

  1. Create Department Wallpapers
    • Create a GPO → Academic Wallpaper.
    • User Configuration → Policies → Administrative Templates → Desktop → Desktop Wallpaper.
    • Edit → Enable → specify wallpaper path → Apply → OK.
    • Repeat for each department with its own wallpaper.
  2. Create GPOs for Restrictions
    • Domain GPOs: Users Policy and Admin Policy.
  3. Users Policy Settings
    • Prohibit access to Control Panel and PC settings.
    • Do not store LAN Manager hash value on next password change.
    • Prevent access to the command prompt.
    • No auto-restart with logged-on users for scheduled updates.
    • Deny access to all removable storage classes.
    • Prohibit user installs.
    • Disable guest account status.
    • Minimum password length: 12 characters.
    • Maximum password age: 42 days.
    • Audit directory service access (success).
    • Account lockout duration: 30 minutes.
    • Account lockout threshold: 3 invalid attempts.
    • Reset account lockout counter after 30 minutes.
    • Remove links/access to Windows Update.
    • Remove network connections from Start Menu.
    • Remove Help menu from Start Menu.
    • Remove Run menu from Start Menu.
    • Remove My Pictures icon from Start Menu.
    • Remove My Music icon from Start Menu.
    • Remove My Network Places icon from Start Menu.
    • Add Logoff option to Start Menu.
    • Remove Task Manager from Ctrl+Alt+Del.
    • Set Internet Explorer browser title bar text to domain name.
    • Prohibit access to LAN connection properties.
    • Prevent access to registry editing tools.
  4. Admin Policy Settings
    • Apply opposite settings where required (e.g., if enabled in User Policy, disable in Admin Policy).
