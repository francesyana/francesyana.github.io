Case Study Part 4

This section presents Case Study Part 4: File Server.

Requirements

  1. All departments/sections should have one public folder (read/write). Domain users cannot access other department folders.
  2. A university public folder where all domain users can access (read/write).
  3. An installer folder containing applications that only the administrator can access.
  4. All folders except the Computer Engineering folder should contain only office documents (.ppt, .doc, .docx, .xls, .pptx, .pdf, etc.).
  5. Information Technology folder should contain all files except executable files.
  6. The storage quota for each folder is 1GB.
  7. Map the shared folder for each client workstation.

Steps

  1. Create Folders
    • Create directories from Part 2 of this case study, plus University and Installer folders.
    • You may also run a script to automate folder creation.
  2. Configure File Services
    • Add roles → File Services → Next → Next.
    • Check all except Windows Server 2003 File Services → Next.
    • DFS Namespaces: NameSpace1 → Next → Next.
    • In credentials, select domain admin username and password → Next → Next.
    • Select local disk → Next until Install → Restart.
  3. Quota Management
    • Start → Administrative Tools → File Service Resource Manager → Quota Management.
    • Quota Templates → Create new quota template.
    • Template Name: 1GB → Space Limit: 1GB → Add.
  4. File Screening Management
    • General: Read/Write, Users: All, Files: Only office documents, File Screen Template: General.
    • IT: Read/Write, Users: Information Technology, File Screen Template: Block Executable Files.
  5. Provision Shared Folders
    • Administrative Tools → Server Manager → Roles → File Services → Share and Storage Management.
    • Create Provision Share → Shared Folder Provision Wizard.
    • Select folder location → Available volumes → C Drive → Next.
    • Check SMB → Next → Configure SMB permissions based on requirements.
    • Apply quota → Apply file screen template → Finish.
  6. University Folder
    • Permissions: Read/Write
    • Users: All Domain Users
    • File Screen Template: General
  7. Installer Folder
    • SMB Share Name: Installer$
    • Permissions: Read/Write
    • Users: Admin only
    • File Screen Template: General
